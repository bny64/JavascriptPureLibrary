# 프로젝트 분석: 업무 관리 시스템

이 문서는 `README.md` 및 초기 코드베이스 조사를 통해 얻은 "업무 관리 시스템" 프로젝트를 요약합니다. 이 시스템은 폐쇄망 환경에서 작동하도록 설계되었으며, 외부 종속성 없이 파일 기반의 업무 관리를 제공합니다.

## 1. 프로젝트 개요

이 프로젝트는 바닐라 Node.js를 백엔드로 사용하고, 일반 JavaScript, HTML, CSS를 프론트엔드로 사용하여 간단한 파일 기반 업무 관리 시스템을 구현합니다. 작업 및 카테고리에 대한 CRUD 작업을 지원하며, 데이터는 JSON 파일(`tasks.json`, `categories.json`)에 영구적으로 저장됩니다. 이 시스템은 Node.js만으로 실행 가능한 경량 아키텍처를 특징으로 합니다.

**강조된 주요 기능:**
- 순수 Node.js (외부 패키지 없음)
- 파일 기반 데이터 저장 (`tasks.json`, `categories.json`)
- 트리 구조 카테고리
- 캘린더 기반 날짜 관리
- 작업 CRUD 작업
- 작업 상태 관리 (대기, 진행 중, 완료, 보류)
- 페이지네이션된 작업 목록
- 반응형 녹색 테마 디자인

## 2. 파일 구조 및 목적

프로젝트 디렉토리에는 다음 주요 파일이 포함되어 있습니다:

-   `api.js`: `server.js`가 요청을 처리하는 데 사용하는 API 엔드포인트 정의 및 로직을 포함할 것으로 예상됩니다.
-   `app-modular.js`: `app.js`의 대체 또는 모듈화된 버전이거나, 더 체계적인 클라이언트 측 로직을 위한 개발 중인 파일일 수 있습니다.
-   `app.js`: UI 상호 작용, 데이터 렌더링 및 백엔드 API와의 통신을 담당하는 기본 클라이언트 측 JavaScript 파일입니다.
-   `categories.json`: JSON 형식으로 카테고리 데이터를 저장합니다. 이 파일은 작업 분류를 위한 간단한 데이터베이스 역할을 합니다.
-   `index.html`: 웹 애플리케이션 사용자 인터페이스의 전체 구조와 레이아웃을 제공하는 메인 HTML 파일입니다.
-   `README-FINAL.md`: 프로젝트 문서의 최종 버전일 가능성이 높습니다.
-   `README-v2.md`: `README.md`의 이전 또는 대체 버전일 수 있습니다.
-   `README.md`: 프로젝트, 기능, 설정 지침 및 사용 가이드에 대한 포괄적인 개요를 제공합니다.
-   `server.js`: Node.js 백엔드 서버입니다. 정적 파일을 제공하고, 작업 및 카테고리 관리를 위한 API 엔드포인트를 제공하며, `tasks.json` 및 `categories.json` 파일에서 읽고 쓰는 방식으로 데이터 영구성을 관리합니다.
-   `styles.css`: 애플리케이션의 시각적 모양, 레이아웃 및 테마를 정의하는 CSS(Cascading Style Sheets)를 포함합니다.
-   `tasks.json`: JSON 형식으로 작업 관련 데이터를 저장합니다. 이 파일은 작업 정보를 위한 간단한 데이터베이스 역할을 합니다.
-   `themes.css`: `styles.css` 외에 애플리케이션을 위한 대체 또는 보조 테마를 정의할 수 있습니다.
-   `todo.txt`: 개발 관련 메모 또는 개발자를 위한 실제 할 일 목록으로 예상되는 간단한 텍스트 파일입니다.
-   `ui.js`: `app.js`에서 분리되어 더 나은 구성을 위해 특정 UI 렌더링 또는 조작 로직을 포함할 것으로 예상됩니다.
-   `utils.js`: 애플리케이션의 여러 부분에서 사용되는 유틸리티 함수 또는 헬퍼 모듈을 포함할 것으로 예상됩니다.

## 3. 주요 기능

### 백엔드 (`server.js`, `api.js`)
-   **API 엔드포인트**: 작업 및 카테고리 관리를 위한 RESTful API를 제공합니다.
-   **데이터 영구 저장**: `tasks.json` 및 `categories.json`에 각각 작업 및 카테고리 데이터를 읽고 씁니다.
-   **정적 파일 제공**: `index.html`, `app.js`, `styles.css` 등을 클라이언트에 제공합니다.

### 프론트엔드 (`index.html`, `app.js`, `ui.js`, `styles.css`, `themes.css`)
-   **헤더**: 카테고리 관리 및 새 작업 추가 버튼을 포함합니다.
-   **캘린더 보기**: 특정 날짜의 작업 상태를 점으로 표시하는 월별 캘린더를 표시합니다. 날짜를 클릭하면 해당 날짜의 작업이 표시됩니다.
-   **선택된 날짜 작업**: 현재 선택된 날짜의 작업 목록을 편집/삭제 옵션과 함께 표시합니다.
-   **모든 작업 목록**: 모든 작업을 페이지네이션(페이지당 5개 항목)하여 시간순으로 표시합니다. 작업을 클릭하면 캘린더의 해당 날짜로 이동합니다.
-   **카테고리 관리**: 계층적 트리 구조로 주 카테고리 및 하위 카테고리의 생성, 편집 및 삭제를 허용합니다.
-   **작업 관리**:
    -   **추가**: 카테고리 선택, 시작/종료 날짜, 상태 및 설명을 설정하여 새 작업을 생성합니다.
    -   **편집**: 기존 작업 세부 정보를 수정합니다.
    -   **삭제**: 시스템에서 작업을 제거합니다.
-   **작업 상태**: 시각적 표시 (대기 - 노란색, 진행 중 - 파란색, 완료 - 녹색, 보류 - 회색).
-   **날짜 탐색**: 캘린더 상호 작용, 오늘 날짜 강조 표시.
-   **반응형 디자인**: 현대적인 카드 디자인과 호버 효과를 갖춘 녹색 그라디언트 테마.

## 4. 시스템 요구사항

-   Node.js (v12 이상)

## 5. 설치 및 실행

1.  모든 핵심 파일 (`server.js`, `index.html`, `styles.css`, `app.js`)이 동일한 디렉토리에 있는지 확인합니다.
2.  서버 실행: `node server.js`
3.  웹 브라우저에서 접속: `http://localhost:3000`

## 6. 중요 고려사항

-   폐쇄망 환경에서 사용하도록 설계되었습니다. 외부 CDN/인터넷이 필요하지 않습니다.
-   여러 동시 사용자와의 데이터 충돌 가능성이 있습니다.
-   `tasks.json` 및 `categories.json`의 정기적인 데이터 백업을 권장합니다.
-   작업을 추가하기 전에 카테고리를 먼저 등록해야 합니다.
-   포트는 `server.js`에서 변경할 수 있습니다 (기본값: 3000).

이 분석은 프로젝트에 대한 기본적인 이해를 제공하여 향후 개발 및 유지 관리를 가능하게 합니다.
